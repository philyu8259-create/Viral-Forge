# Alibaba Cloud FC + Tablestore Deployment

ViralForge backend should run on Alibaba Cloud Function Compute, with Tablestore as the production data store.

## Runtime Shape

- Function Compute: built-in Node.js 20 web function with ZIP upload
- Runtime package: generated by `npm run deploy:package-fc-runtime`
- Request handler: `index.handler`
- Public access: HTTP trigger or custom domain
- Persistent data: Tablestore, not SQLite
- Provider mode: `AI_PROVIDER_MODE=china_live`

The backend must expose:

```text
GET /health
GET /api/providers/status
POST /api/content/generate
POST /api/poster/background
POST /api/app-store/notifications/v2
```

## Tablestore Table

Create one wide table for the MVP:

```text
Table name: viralforge_records
Primary key:
  pk STRING
  sk STRING
```

Recommended row keys:

```text
pk=user#<user_id>, sk=quota
pk=user#<user_id>, sk=brand
pk=user#<user_id>, sk=project#<project_id>
pk=user#<user_id>, sk=subscription#<original_transaction_id>
pk=notification, sk=<notification_uuid>
```

The application stores JSON payloads in regular attribute columns. This keeps the first beta schema simple and avoids SQLite/NAS persistence in FC.

## Environment

Use `deploy/fc/fc.env.example` as the required variable list. Do not commit real values.

Before deploying, validate the local shell environment:

```sh
cd backend
set -a
source deploy/fc/fc.env
set +a
npm run deploy:check-fc-env
```

## Deploy Steps

1. Run `npm install`.
2. Run `npm run deploy:package-fc-runtime`.
3. Create an FC web function with built-in runtime `Node.js 20`.
4. Select ZIP package upload and upload `dist/viralforge-fc-runtime.zip`.
5. Set the request handler to `index.handler`.
6. Add an HTTP trigger or bind a custom HTTPS domain.
7. Configure all variables from `deploy/fc/fc.env.example`.
8. Grant the function role access to the Tablestore instance/table, or provide `TABLESTORE_ACCESS_KEY_ID` and `TABLESTORE_ACCESS_KEY_SECRET`.
9. Smoke test:

```sh
curl https://YOUR_BACKEND_DOMAIN/health
curl https://YOUR_BACKEND_DOMAIN/api/providers/status
curl https://YOUR_BACKEND_DOMAIN/api/app-store/status
```

After the public HTTPS URL works, set the iOS Release `BACKEND_BASE_URL` to that URL and rebuild the TestFlight archive.

## Function Role Permission

Prefer FC context credentials over static AccessKeys. In the FC advanced configuration, enable a function role whose trusted service is Function Compute, then attach either the managed `AliyunOTSFullAccess` policy for the MVP or a custom least-privilege policy for this table:

```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "ots:GetRow",
        "ots:PutRow",
        "ots:DeleteRow",
        "ots:GetRange"
      ],
      "Resource": "acs:ots:cn-hangzhou:<account_id>:instance/ai-accountant-cu/table/viralforge_records"
    }
  ]
}
```

Keep `USE_CONTEXT_CREDENTIALS=1` in the function environment after the role is granted. A healthy deployment must return `200` for both `/health` and `/api/quota`; if `/health` works but `/api/quota` returns `403`, the function role is missing Tablestore data access.
